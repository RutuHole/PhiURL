Common Questions

Everything you need to know about PhiUrl and how it works.

How does PhiUrl detect phishing?
We use a logistic regression model trained on character-level TF-IDF n-grams (3–5 chars). The model learns which character sequences in a URL are statistically associated with phishing — suspicious TLDs, keyword patterns, and structural anomalies.
Does it need to visit the URL?
No. PhiUrl analyzes only the URL string itself — no HTTP requests are made to the target site. You can safely scan suspicious links without triggering them.
Is there a backend I need to run?
Yes — the site connects to a Flask backend (app.py) on localhost:5000. If offline, the nav badge shows Demo Mode and results are generated locally. Run python app.py to use your real ML model.
